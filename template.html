Open weights just reached the closed frontier — as Washington keeps Anthropic's flagships dark

Two open-weight models released this month now sit at the top of open coding benchmarks and edge a Western closed flagship. MiniMax M3 (Jun 1) pairs frontier-tier coding with a 1M-token context and native multimodality, posting 59.0% on SWE-Bench Pro — reportedly above GPT-5.5 and Gemini 3.1 Pro — plus 66.0% Terminal-Bench 2.1 and 74.2% MCP Atlas (Hugging Face, Kilo). Days later, Z.AI's GLM-5.2 (753B total / 40B active, 1M context) went further — 62.1% SWE-Bench Pro and 81.0% Terminal-Bench 2.1, again above GPT-5.5's ~58.6% (Kilo). The strategic read writes itself: at the exact moment the US has switched off Anthropic's Fable 5 / Mythos 5 for foreign nationals (today's lead Developing thread, Day 12), open weights that anyone can self-host have closed the gap to the closed labs on the workload enterprises care most about — coding. The catch is downstream: these gains land in agent stacks where prompt injection is now described as a permanent, not patchable, flaw (see New #2).

So what: a self-hostable model that matches GPT-5.5 on coding changes the build-vs-buy and the export-exposure math for every platform team — pull MiniMax M3 / GLM-5.2 into your eval harness this week and price what an open-weight fallback would actually cost you to run.

IMPACT 8/10#2 POLICY · SECURITY

The agent-security reckoning: prompt injection called a permanent, not patchable, flaw

As agents go GA across the stack, the security bill arrives. Prompt injection is now OWASP's #1 agentic-AI threat, mapped to six of ten categories, and researchers increasingly frame it as a structural flaw rather than a bug to patch (Help Net Security, TechTimes). Recent proof: a LiteLLM supply-chain backdoor sat on PyPI for three hours with ~47,000 downloads (the gateway for CrewAI, DSPy, GraphRAG), and Microsoft showed a Semantic Kernel path turning a single prompt into host-level RCE (Microsoft Security). Simon Willison's "lethal trifecta" — private data + untrusted content + external comms — is the live failure mode.

If you're shipping agents, treat prompt injection as a permanent threat model, not a backlog ticket — isolate the trifecta, pin your framework deps, and assume any untrusted input can become code.

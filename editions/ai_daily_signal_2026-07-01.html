■ Today's Lead · New · Security · Wednesday deep beat

A decades-old Bash trick just walked past the safety guardrails in almost every open-source AI coding agent.

Security firm Adversa AI disclosed "GuardFall" (~Jun 29–30) — a structural flaw, not a single bug: AI coding and computer-use agents inspect a shell command as plain text for danger, but Bash rewrites that command (dequoting, variable/glob expansion) before it runs, so the string the guardrail approved is not the string that executes. In testing it bypassed the safety checks of 10 of 11 popular open-source agents (Hermes, OpenCode, Roo-code and others; only "Continue" held), turning a poisoned repo, README or Makefile into supply-chain remote code execution that can exfiltrate SSH/AWS credentials or wipe a developer's environment (The Hacker News, SecurityWeek). It lands alongside a fresh cluster of unpatched critical CVEs in AI dev tooling (New #5) — the same week AWS is standing up a $1B army of engineers to push exactly these agents into production (New #2).

So what: the guardrail on your coding agent is not a security boundary. If your teams run open-source or auto-executing agents against untrusted repos, treat that as arbitrary code execution today — sandbox a throwaway $HOME, kill --auto-exec / "dangerously-skip-permissions," and stop pointing agents at code you haven't vetted.

The mechanism. An AI agent proposes a shell command; a guardrail scans the literal text for dangerous patterns (rm -rf, credential paths, network calls) and approves it. But Bash performs quote-removal, parameter expansion and word-splitting after that check — so a command that looks benign as text can resolve, at execution time, into something destructive. The guardrail and the shell disagree about what the command "is," and the shell wins.

The blast radius. Adversa reports 10 of 11 tested open-source coding / computer-use agents were bypassable; only "Continue" resisted. Because agents routinely read a repository's own files (README, Makefile, task descriptions) as instructions, a booby-trapped public repo becomes an indirect-prompt-injection + guardrail-bypass chain: clone, "let the agent set it up," and it runs attacker code — a classic supply-chain vector, now against the tools developers trust most (The Hacker News, SecurityWeek).

Why it's the lead. This is the Wednesday security deep beat and the freshest strongly-sourced story of the day — but it matters beyond the beat: the entire 2026 enterprise pitch (ServiceNow, SAP, Microsoft, and now AWS's $1B forward-deployed-engineer org) is "put autonomous coding/task agents into production." GuardFall says the safety layer those deployments lean on is porous by construction, and it arrives in a week already thick with unpatched AI-tooling CVEs (Flowise 9.9, Crawl4AI 9.8, Langflow actively exploited).

Caveats & watch next. This is a class of open-source agents, not a named CVE with a patch; the fix is architectural (execute-then-verify, hardened sandboxes, no auto-exec on untrusted input) rather than a version bump. Watch which agent projects ship dequoting-aware guardrails, whether commercial IDE agents (Cursor, Copilot, Codex) are shown vulnerable to the same pattern, and how enterprise agent-governance vendors respond.
